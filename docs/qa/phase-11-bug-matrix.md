# Phase 11 - Comprehensive QA Bug Matrix

هذا التقرير مبني على فحص وظيفي/تقني للكود الحالي + تشغيل `lint/build` + مراجعة تدفقات المستخدم الحرجة.

## Critical

### 1) تسجيل الخروج غير فعّال من الشريط العلوي
- **الحالة:** تم إصلاحه.
- **الخطورة:** Critical
- **خطوات إعادة الإنتاج (قبل الإصلاح):**
  1. تسجيل الدخول.
  2. فتح قائمة الحساب من `Topbar`.
  3. الضغط على "تسجيل الخروج".
  4. لا يحدث إنهاء جلسة فعلي.
- **السبب الجذري:** عنصر القائمة بلا handler يستدعي `signOut`.
- **الإصلاح المطبق:** ربط `signOut(auth)` + redirect + رسائل نجاح/فشل.

### 2) تسجيل الخروج في بوابة العميل لا ينهي الجلسة
- **الحالة:** تم إصلاحه.
- **الخطورة:** Critical
- **خطوات إعادة الإنتاج (قبل الإصلاح):**
  1. فتح `/client-portal`.
  2. الضغط على أيقونة الخروج.
  3. الانتقال بدون logout فعلي.
- **السبب الجذري:** كان هناك `navigate('/')` فقط.
- **الإصلاح المطبق:** `signOut(auth)` ثم redirect + toast.

## High

### 3) أزرار إجراءات متعددة Placeholder بدون أثر
- **المكان:** عدة views (مثل Documents/Tasks/Portal sections)
- **الأثر:** المستخدم قد يعتقد أن الميزة تعمل بينما هي واجهة فقط.
- **التوصية:** تمييز الأزرار غير المكتملة بعلامة "قريبًا" أو تعطيلها مؤقتًا.

### 4) عدم وجود E2E tests للتدفقات الحرجة
- **الأثر:** صعوبة ضمان الاستقرار بعد كل تغيير.
- **التوصية:** إضافة smoke suite للتدفقات: login -> dashboard -> cases -> upload -> logout.

## Medium

### 5) الاعتماد على mock data في أجزاء واسعة من النظام
- **الأثر:** فجوة بين سلوك الديمو وسلوك الإنتاج.
- **التوصية:** استكمال المرحلة 04/05 على باقي الموديولات تدريجيًا.

### 6) تحذيرات bundle size ما زالت موجودة
- **الأثر:** تحميل أولي أبطأ على شبكات ضعيفة.
- **التوصية:** manualChunks + تحميل مؤجل لمكتبات الرسوم الثقيلة.

## Low

### 7) بعض المسارات لا تعرض Empty/Error states موحدة
- **الأثر:** تفاوت UX عند فشل الشبكة.
- **التوصية:** مكوّن موحّد لحالات loading/error/empty في جميع الصفحات.

---

## Prioritized Remediation Plan

1. **Critical (فوري):** إصلاح logout (تم).
2. **High:** إغلاق الفجوات الوظيفية الظاهرة للمستخدم (placeholder actions / smoke E2E).
3. **Medium:** استكمال data migration وتوحيد حالات الخطأ.
4. **Low:** تحسينات الأداء والاتساق البصري.
