# Logging Policy (Security Baseline)

## الهدف
تقليل تسريب البيانات الحساسة في السجلات مع الحفاظ على قابلية التتبع الفني.

## ممنوع تسجيله في logs
- عناوين البريد الإلكتروني الكاملة.
- أرقام الهوية الوطنية/الإقامة.
- أرقام السجل التجاري والرقم الضريبي.
- نصوص القضايا أو المستندات القانونية الخام.
- مفاتيح API أو رموز الجلسات أو Access Tokens.

## مسموح تسجيله
- `requestId` فريد لكل خطأ.
- نوع العملية (`operationType`) ومسار منطقي مختصر.
- معرف المستخدم `uid` فقط.
- `tenantId` عند توفره.
- نوع الخطأ (Error Name/Code) بدون payload.

## قواعد التنفيذ
1. أي خطأ يمر عبر نقطة مركزية للـ sanitization.
2. واجهة المستخدم تعرض رسالة عامة مع `requestId` فقط.
3. يمنع طباعة raw exceptions القادمة من مزودات AI أو DB.
4. أي debugging مؤقت يجب حذفه قبل الدمج.

## Checklist سريعة قبل الدمج
- [ ] لا يوجد `console.log` لبيانات حساسة.
- [ ] لا يوجد `console.error(error)` على كائنات خام تحتوي payload.
- [ ] كل أخطاء Firestore تحتوي `requestId`.
- [ ] رسائل المستخدم لا تتضمن stack trace أو JSON داخلي.
