# ⚖️ ملف (malaf.site) - منصة الإدارة القانونية الذكية

**ملف (malaf.site)** هي منصة برمجية خدمية (SaaS) مصممة لمكاتب المحاماة في العالم العربي. المنصة تدمج بين الذكاء الاصطناعي وإدارة القضايا لتقديم حل تقني شامل يربط المحامي بموكليه، وينظم الشؤون المالية، ويحمي البيانات بأعلى المعايير الأمنية.

---

## 🚀 الرؤية والهدف (The Vision)
تهدف المنصة إلى رقمنة قطاع المحاماة العربي بالكامل، والانتقال من الإدارة الورقية العشوائية إلى لوحة تحكم رقمية واحدة. حيث تقوم المنصة بأتمتة المهام الروتينية، صياغة العقود بالذكاء الاصطناعي، ومتابعة قضايا التنفيذ المالي، ليتفرغ المحامي للترافع الاستراتيجي.

---

## ✨ الميزات الحصرية التنافسية (Key Features)

### 1. 🤖 المحامي الذكي (AI Legal Assistant)
نظام ذكاء اصطناعي مدمج يعمل بـ `Gemini API` ويحتوي على مسار احتياطي للعمل دون اتصال بالخادم (Fallback Mock System):
- **المستشار القانوني الآلي:** يقدم استشارات مبدئية استناداً إلى الأنظمة القانونية العربية.
- **صانع العقود الذكي:** توليد مسودات عقود قانونية معتمدة بضغطة زر.
- **محلل الوثائق:** تحليل العقود والمستندات لاستخراج نقاط القوة والضعف والمخاطر.

### 2. 🛡️ بنية أمنية مصرفية (Bank-Grade Security)
- **تشفير AES-256:** جميع البيانات الحساسة (الهويات، المبالغ المالية، ملفات القضايا) يتم تشفيرها وفك تشفيرها محلياً في المتصفح قبل إرسالها إلى قواعد البيانات.
- **العزل الشامل للمستأجرين (Multi-Tenancy Isolation):** كل مكتب محاماة (Tenant) لديه بيئة بيانات معزولة تماماً في PostgreSQL عبر Supabase، مع Row Level Security (RLS) صارمة تمنع تداخل البيانات بين المكاتب.
- **سجلات الأمان والتدقيق (Audit Logs):** تتبع دقيق لأي إجراء مالي أو عرض لمعلومات حساسة مع تسجيل (وقت الدخول، واسم الموظف).

### 3. 💼 إدارة مالية وتنفيذ (Financial & Enforcement)
- **إدارة حسابات الأمانات:** نظام شفاف لإدارة مبالغ العملاء ومقدم الأتعاب وفصلها عن حسابات المكتب.
- **إدارة التنفيذ المالي:** نظام متكامل لفرز أوامر التنفيذ، إشعارات الـ 46، وأوامر الحجز والمنع، مع تتبع آلي للمبالغ المحصلة والمتبقية.

### 4. ⚡ بنية تحتية سحابية متقدمة (Modern Infrastructure)
- **PostgreSQL + Supabase:** قاعدة بيانات علائقية قوية مع فهارس متقدمة، Pagination سريع، واستعلامات معقدة.
- **تكامل مع هيئة الزكاة (ZATCA-Ready):** جاهزية البنية التحتية لإصدار فواتير إلكترونية متوافقة مع المتطلبات الضريبية.

## 🛠️ التقنيات المستخدمة (Tech Stack)

| التقنية (Technology) | الغرض (Purpose) |
| :--- | :--- |
| **React 18 + Vite** | واجهة مستخدم سريعة جداً وتفاعلية (Frontend) |
| **Tailwind CSS + shadcn/ui** | تصميم عصري، سلس، ومريح للعين مع دعم الوضع الليلي (Dark Mode) |
| **Zustand** | إدارة حالة التطبيق محلياً لضمان السرعة الفائقة وتقليل التحميل على السيرفر |
| **Supabase (Auth + PostgreSQL + Storage)** | مصادقة، قاعدة بيانات علائقية، وتخزين ملفات سحابي |
| **Crypto-JS (AES-256)** | تشفير البيانات الحساسة من طرف إلى طرف (End-to-End Encryption) |
| **Zod** | تحقق دقيق من صحة مدخلات المستخدم قبل الإرسال لمنع الاختراقات |

## 💻 طريقة التشغيل المحلية (Local Development)

بالنسبة للمطورين أو المستثمرين الراغبين في معاينة المشروع محلياً، يمكن اتباع الخطوات التالية:

**1. تنصيب الحزم:**
\`\`\`bash
npm install
\`\`\`

**2. إعداد متغيرات البيئة:**
قم بإنشاء ملف `.env` في الجذر، وأضف المفاتيح التالية (احصل عليها من لوحة تحكم Supabase):
\`\`\`env
VITE_SUPABASE_URL=https://your-project.supabase.co
VITE_SUPABASE_ANON_KEY=your_anon_key
VITE_ENCRYPTION_KEY=your_secret_encryption_key
GEMINI_API_KEY=your_gemini_api_key
\`\`\`

**3. تشغيل الواجهة الأمامية (Frontend):**
\`\`\`bash
npm run dev
\`\`\`

**4. لتشغيل الخادم الخلفي للذكاء الاصطناعي (اختياري):**
\`\`\`bash
# في نافذة طرفية منفصلة
node server.js
\`\`\`
*(ملاحظة: النظام يحتوي على ميزة "البديل الذكي المحلي" الذي يضمن عمل ميزات الذكاء الاصطناعي وتوليد العقود بأمثلة واقعية حتى في حال عدم تشغيل الخادم).*
